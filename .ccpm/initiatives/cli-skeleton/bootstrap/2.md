# Task: Bootstrap ebless package with index CLI and tests

## Description

Stand up the entire `ebless` Python package skeleton in one go: `uv`-managed project metadata, package layout, Click-based CLI with `index` subcommand wired to a stub indexer, pytest test suite, and updated README. After this task, `uv sync` produces a working `ebless` on PATH and `uv run pytest` exits 0.

## Acceptance Criteria
- [ ] `pyproject.toml` exists at the repo root with PEP 621 metadata: `name = "ebless"`, `requires-python = ">=3.11"`, runtime dep `click>=8.1`, dev dep `pytest>=8`, `[project.scripts] ebless = "ebless.cli:cli"`, `[build-system]` using `hatchling`.
- [ ] `uv.lock` is generated by `uv sync` (or `uv lock`) and committed.
- [ ] `ebless/__init__.py` exists (may be empty or expose only a `__version__`).
- [ ] `ebless/cli.py` defines a Click `cli` command group and an `index` subcommand. The subcommand accepts a positional `path` argument validated via `click.Path(exists=True, file_okay=False, dir_okay=True, path_type=pathlib.Path)`, and calls `ebless.indexer.index_books(path)`.
- [ ] `ebless/indexer.py` defines `index_books(path: pathlib.Path) -> None` that prints exactly `would index <path>` (using the path argument's string form) to stdout and returns.
- [ ] `tests/test_cli.py` exists and covers, with assertions, all three acceptance paths from the initiative's US-2 plus the US-3 invocation assertion:
  - happy path: `ebless index <tmp_path>` exits 0 and stdout contains `would index <tmp_path>`
  - missing-path error: `ebless index <nonexistent>` exits non-zero
  - stub invocation: a test confirms `index_books` is the function the CLI calls (e.g. by asserting on its observable output via `CliRunner`, or by importing it directly and asserting its stdout — pick one approach, do not duplicate the assertion across both)
- [ ] `uv sync` from a clean state exits 0 and makes `ebless` invocable as `uv run ebless`.
- [ ] `uv run ebless --help` exits 0 and lists `index`.
- [ ] `uv run ebless index <some-existing-dir>` prints `would index <some-existing-dir>` and exits 0.
- [ ] `uv run ebless index /does/not/exist` exits non-zero.
- [ ] `uv run pytest` exits 0 with all tests passing.
- [ ] `README.md` Install section uses `uv sync` (or `uv pip install -e .`); Usage section's `ebless index` example matches the implemented behavior. Replace any "Follow the installation instructions for your platform." placeholder text.

**`ebless/indexer.py`** — keep it minimal:
```python
from pathlib import Path

def index_books(path: Path) -> None:
    print(f"would index {path}")
```

The signature `(path: Path) -> None` is the contract for the future real indexer — do not change it without coordinating downstream.

**`ebless/cli.py`** — Click group + subcommand:
```python
import pathlib
import click
from ebless.indexer import index_books

@click.group()
def cli() -> None:
    """ebless — eBook LLM Empowered Semantic Search."""

@cli.command("index")
@click.argument(
    "path",
    type=click.Path(exists=True, file_okay=False, dir_okay=True, path_type=pathlib.Path),
)
def index_cmd(path: pathlib.Path) -> None:
    """Index every supported eBook under PATH."""
    index_books(path)
```

**Tests** — use Click's `CliRunner`:
```python
from click.testing import CliRunner
from ebless.cli import cli

def test_index_happy_path(tmp_path):
    result = CliRunner().invoke(cli, ["index", str(tmp_path)])
    assert result.exit_code == 0
    assert f"would index {tmp_path}" in result.output

def test_index_missing_path(tmp_path):
    missing = tmp_path / "does-not-exist"
    result = CliRunner().invoke(cli, ["index", str(missing)])
    assert result.exit_code != 0
```

Add one more test that exercises `index_books` directly (e.g. via `capsys`) to assert the stub's contract independent of the CLI wiring — this gives a regression net for the future real indexer without coupling to Click.
