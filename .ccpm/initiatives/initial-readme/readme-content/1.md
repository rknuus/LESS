# Task: Write README.md

## Description

Author the project's first user-facing `README.md` at the repository root. Target audience is a prospective LESS user who needs to understand what LESS does, install it, and run their first index/search against an eBook directory without reading any source code.

## Acceptance Criteria

- [ ] `README.md` exists at the repository root.
- [ ] Sections appear in this order: *What is LESS*, *Why semantic search*, *Supported formats*, *Setup*, *Usage*, *How it works*.
- [ ] *What is LESS* is at most one paragraph and names the supported file formats and the semantic-search value proposition.
- [ ] *Setup* lists prerequisites (runtime, vector DB, embedding model/API), installation commands, and a verification step.
- [ ] *Usage* shows runnable example commands for both indexing a directory and querying it, with placeholder paths the reader can substitute.
- [ ] *How it works* gives a brief, non-architectural explanation of the embedding + vector-similarity flow.
- [ ] Markdown only — no badges, no HTML, no embedded images.
- [ ] Whole file is under ~200 lines.
- [ ] All Markdown links use proper `[text](url)` syntax (no raw URLs in body text).

## Technical Details

- File location: `README.md` at the repository root (not under `.claude/` or `.ccpm/`).
- The CLI surface (e.g., `index`, `search`) is not yet implemented in this repo. Write the *Setup* and *Usage* sections against an intended interface, with command names and flags chosen to be plausible and easy to revise. Use placeholder values like `<path/to/books>` and `<query>` rather than committing to specific real arguments.
- Keep examples generic so small interface changes do not invalidate the README.
